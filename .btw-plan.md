## Architecture

- One global overlay, shows one active slot at a time
- Header/tab strip shows other slots and their status
- Many slot states — each slot = one independent BTW thread with own AgentSession
- Per-slot persistence with slotId
- Fire-and-forget execution per slot

## Data Structures

```ts
type PersistedBtwEntry = BtwDetails & { slotId: string };

type BtwSlotState = {
  id: string;
  title: string;
  thread: BtwDetails[];
  draft: string;
  status: string;
  busy: boolean;
  pendingQuestion: string | null;
  pendingAnswer: string;
  pendingError: string | null;
  pendingToolCalls: ToolCallInfo[];
  runtime: SideSessionRuntime | null;
  createdAt: number;
  updatedAt: number;
};

type BtwStore = {
  slots: Map<string, BtwSlotState>;
  order: string[];
  activeSlotId: string | null;
};
```
