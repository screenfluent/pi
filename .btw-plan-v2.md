## Architecture
- One global overlay, shows one active slot at a time
- Tab/header strip shows other slots and status
- Per-slot state instead of globals
- Slot-aware persistence via append-only event log
- Serialize slot execution (only one slot runs at a time — safe with codingTools)
- Esc hides overlay only (no inject/summary prompt on close)
- Widget status when overlay hidden: "BTW: 2 running, 1 ready — /btw to open"

## Persisted event types
- `btw-slot-entry` — completed exchange with slotId
- `btw-slot-archive` — marks a slot as archived
- Legacy compat: `btw-thread-entry` and `btw-thread-reset` (no slotId → "slot-legacy")

## Data structures

```ts
type PersistedBtwEntry = BtwDetails & { slotId: string };
type PersistedBtwArchive = { slotId: string; timestamp: number };

type BtwPendingState = {
  question: string | null;
  answer: string;
  error: string | null;
  toolCalls: Map<string, ToolCallInfo>; // keyed by toolCallId
  statusText: string;
};

type BtwSlotState = {
  id: string;
  title: string;              // from first question
  thread: BtwDetails[];       // completed exchanges only
  draft: string;              // ephemeral
  busy: boolean;              // ephemeral
  pending: BtwPendingState;   // ephemeral
  runtime: SideSessionRuntime | null; // ephemeral
  createdAt: number;
  updatedAt: number;
};

type BtwStore = {
  slots: Map<string, BtwSlotState>;
  order: string[];
  activeSlotId: string | null;
};
```

## Command semantics

- `/btw` — open overlay on active slot; create empty slot if none
- `/btw <question>` — create NEW slot, start prompt, switch overlay to it
- `/btw:new [question]` — create fresh slot, optionally start
- `/btw:inject [instructions]` — inject active slot thread to main session, archive slot
- `/btw:summarize [instructions]` — summarize active slot, inject summary, archive slot
- `/btw:archive` — archive active slot without injection

## Overlay behavior
- One overlay, active slot transcript only
- Tab strip: `[1] auth bug ⏳  [2] refactor ✓  [3] notes`
- Per-slot draft preserved on switch
- Esc = hide only
- setWidget summary when hidden

## Restore logic (event replay, NOT grouping)
1. Clear all in-memory slots/runtimes
2. Scan current branch entries in order
3. Legacy entries after last legacy reset → "slot-legacy"
4. New entries: btw-slot-entry → create/append slot, btw-slot-archive → remove slot
5. activeSlotId = most recently updated surviving slot
6. Reset all ephemeral state (busy=false, pending cleared, runtime=null)

## Session/branch lifecycle
Abort all running slot sessions before any session/branch change:
- session_before_switch, session_before_tree, session_before_fork, session_shutdown

Restore after:
- session_start, session_switch, session_tree, session_fork

Dismiss overlay on session changes.

## Concurrency policy
SERIALIZE slot execution. Only one slot prompt at a time.
Queue others. This avoids file races with codingTools.
